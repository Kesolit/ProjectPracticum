# Системный анализ и проектирование (Спринт 1)

## 1. Определение MVP (Границы проекта)
[cite_start]В рамках первой итерации (MVP) реализуются следующие ключевые возможности[cite: 27, 32]:
* [cite_start]**Редактор**: создание структуры портфолио из обязательных секций[cite: 34].
* [cite_start]**Секции**: "О себе", "Навыки", "Проекты", "Опыт", "Образование", "Ссылки", "Контакты"[cite: 34].
* [cite_start]**Механика**: добавление, удаление и изменение порядка блоков[cite: 16, 80, 81].
* [cite_start]**Публикация**: сохранение черновика и генерация публичной ссылки вида `/u/<slug>`[cite: 36, 37].

## 2. User Stories (Пользовательские сценарии)
[cite_start]Для реализации функционала конструктора определены следующие истории[cite: 65]:
* [cite_start]**Добавление**: Как пользователь, я хочу добавлять новые секции из списка шаблонов, чтобы наполнять портфолио контентом[cite: 80].
* [cite_start]**Сортировка**: Как пользователь, я хочу перемещать блоки вверх и вниз, чтобы выстроить логичную структуру самопрезентации[cite: 81].
* [cite_start]**Темизация**: Как пользователь, я хочу переключаться между 1–2 визуальными темами, чтобы настроить внешний вид страницы[cite: 35, 82].
* [cite_start]**Просмотр**: Как пользователь, я хочу видеть превью страницы перед публикацией, чтобы убедиться в отсутствии ошибок[cite: 83].

## 3. Модель данных (API-контракт)
[cite_start]Проектируемая сущность портфолио включает[cite: 14, 67]:
* **Portfolio**: `id`, `userId`, `themeId`, `status` (draft/published).
* **Sections**: массив объектов с полями `id`, `type`, `order`, `content` (JSON-данные блока).
